Overview

This document provides comprehensive information about error handling in the AEAT VERI*FACTU webservice, including error codes, validation rules, and troubleshooting guidance.

HTTP Status Codes

Standard HTTP Status Codes

Success Codes

  • 200 OK: Successful operation
  • 202 Accepted: Operation accepted for processing

Client Error Codes

  • 400 Bad Request: Invalid request format or data
  • 401 Unauthorized: Authentication failure
  • 403 Forbidden: Access denied
  • 404 Not Found: Endpoint not found

Server Error Codes

  • 500 Internal Server Error: Server processing error
  • 502 Bad Gateway: Gateway error
  • 503 Service Unavailable: Service temporarily unavailable
  • 504 Gateway Timeout: Request timeout

AEAT-Specific Status Codes

422

Unprocessable Entity - Business logic validation failed

429

Too Many Requests - Rate limit exceeded

451

Unavailable For Legal Reasons - Legal restrictions

SOAP Faults

SOAP Fault Structure

<soap:Fault xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Code>
    <soap:Value>soap:Client</soap:Value>
    <soap:Subcode>
      <soap:Value>aeat:ValidationError</soap:Value>
    </soap:Subcode>
  </soap:Code>
  <soap:Reason>
    <soap:Text xml:lang="en">Invalid NIF format</soap:Text>
  </soap:Reason>
  <soap:Detail>
    <aeat:Error>
      <aeat:CodigoError>E001</aeat:CodigoError>
      <aeat:DescripcionError>Invalid NIF format</aeat:DescripcionError>
      <aeat:Timestamp>2024-01-15T10:30:30</aeat:Timestamp>
    </aeat:Error>
  </soap:Detail>
</soap:Fault>

Fault Codes

  • soap:Client: Client-side error
  • soap:Server: Server-side error
  • soap:VersionMismatch: SOAP version mismatch
  • soap:MustUnderstand: Must understand header not processed

AEAT Error Codes

  • E001-E099: Validation errors
  • E100-E199: Authentication errors
  • E200-E299: Business logic errors
  • E300-E399: System errors

Common Error Codes

Validation Errors (E001-E099)

  • E001: Invalid NIF format
  • E002: Invalid invoice number
  • E003: Invalid amount format
  • E004: Invalid date format
  • E005: Missing required field

Authentication Errors (E100-E199)

  • E101: Certificate not found
  • E102: Certificate expired
  • E103: Certificate not valid
  • E104: Certificate not authorized
  • E105: Certificate chain invalid

Business Logic Errors (E200-E299)

  • E201: Invoice already exists
  • E202: Invoice not found
  • E203: Invoice already cancelled
  • E204: Invalid invoice type
  • E205: Invalid tax calculation

System Errors (E300-E399)

  • E301: Service temporarily unavailable
  • E302: Database connection error
  • E303: Rate limit exceeded
  • E304: Request timeout
  • E305: Internal server error

Troubleshooting Guide

Common Issues and Solutions

Certificate Issues

  • • Verify certificate is properly installed
  • • Check certificate expiration date
  • • Ensure certificate is from FNMT
  • • Verify certificate has proper permissions

Validation Errors

  • • Check NIF format and validity
  • • Verify invoice number format
  • • Ensure all required fields are present
  • • Validate date formats (YYYY-MM-DD)

Network Issues

  • • Check internet connectivity
  • • Verify firewall settings
  • • Check proxy configuration
  • • Verify SSL/TLS settings

Rate Limiting

  • • Implement exponential backoff
  • • Reduce request frequency
  • • Use batch operations when possible
  • • Monitor rate limit headers
